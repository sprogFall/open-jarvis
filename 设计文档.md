
智能 Agent 分布式管理系统 (Omni-Agent) 设计文档
一、 系统功能清单 (Functional Requirements)
系统需要具备以下核心能力，以实现“感知-决策-执行”的闭环：
1. 核心 Agent 能力
 * 意图解析： 将模糊的自然语言（如“优化下服务器性能”）拆解为具体步骤。
  * 自主循环 (ReAct)： 类似 Claude Code，Agent 能够根据上一步的执行结果（Observation）自主决定下一步动作。
   * 人类在环 (Human-in-the-loop)： 涉及敏感操作（删除、修改配置、高能耗）时自动挂起，等待授权。
   2. 预设技能集 (Built-in Skills)
    * 文件系统管理： 远程读取日志、配置文件，搜索特定后缀文件，甚至进行文件内容的智能总结。
     * 应用与进程管理： 查看系统负载、重启特定服务、监控应用运行状态。
      * 容器管理 (Docker)： 镜像拉取、容器启停、日志查看、自动清理闲置容器。
       * 智能家居控制 (IoT)： 对接本地 Home Assistant 或直接控制设备 IP，实现灯光、温控、窗帘的自动化。
       3. 连接与安全功能
        * 内网穿透直连： 客户端主动向网关发起长连接，无需公网 IP。
         * 离线审批： APP 关闭或手机掉线时，任务状态在网关持久化，用户再次上线后可继续处理。
         二、 系统架构设计
         系统分为三层：控制层 (APP)、中转层 (Gateway)、执行层 (Client)。
         1. 技术栈选型
         | 组件 | 技术实现 | 关键作用 |
         |---|---|---|
         | APP | Flutter | 提供跨平台一致的 UI，处理任务审批流与实时日志显示。 |
         | Gateway | FastAPI (Python) | 维护任务状态机，管理 WebSocket 路由，存储未处理的任务申请。 |
         | Client | LangGraph (Python) | 运行 Agent 状态机，维护本地 Checkpoint（任务快照）。 |
         | Database | SQLite (Client & Gateway) | 客户端存任务快照，网关存审批队列。 |
         | 推送服务 | FCM (Firebase) / JPush | 当 APP 不在后台时，通过系统通知唤起用户审批。 |
         三、 “技能 (Skills)” 插件化设计
         为了让 Agent 具备无限扩展的能力，所有功能都应以 Skill 形式存在。
         1. Skill 的定义标准
         每个 Skill 都是一个独立的 Python 包，包含以下核心要素：
          * Tools: 使用 @tool 装饰的具体函数，包含详细的 Docstring（供 AI 理解何时调用）。
           * Sandbox: 执行环境限制，例如文件操作 Skill 只能访问指定的 /data 目录。
           2. 典型 Skill 示例 (Docker Skill)

           from langchain.tools import tool

           class DockerSkill:

                                                                                   四、 离线审批与状态流转机制
                                                                                   这是解决“APP 关闭后如何继续”的关键逻辑。
                                                                                   1. 任务状态机
                                                                                   网关负责维护每个 task_id 的生命周期：
                                                                                    * RUNNING: Client 正在自主思考或执行低风险指令。
                                                                                     * AWAITING_APPROVAL: Client 命中 interrupt，将状态上报，网关将此状态存入 DB 并发送 Push。
                                                                                      * APPROVED / REJECTED: 用户在 APP 端操作后，网关更新状态并通知 Client。
                                                                                       * RESUMING: Client 收到信号，从本地 SQLite 加载旧状态并继续。
                                                                                       2. 通信协议 (WebSocket JSON)
                                                                                       当 Client 需要审批时，发送：
                                                                                       {
                                                                                         "type": "INTERRUPT_REQUEST",
                                                                                           "task_id": "9527",
                                                                                             "checkpoint_id": "cp_001",
                                                                                               "command": "docker rm -f database_prod",
                                                                                                 "reason": "用户要求清理环境，该容器占用资源较多"
                                                                                                 }

                                                                                                 五、 APP (Flutter) 实现重点
                                                                                                 1. 任务恢复逻辑
                                                                                                 APP 启动时，首先请求网关接口：GET /tasks/pending_approvals。
                                                                                                  * 如果是空，进入普通聊天模式。
                                                                                                   * 如果有数据，弹出 “任务继续执行” 的引导卡片，展示挂起的指令和 AI 的思考过程。
                                                                                                   2. UI 交互设计
                                                                                                    * 代码预览组件： 使用 flutter_highlight 渲染即将执行的 Shell 或 Python 代码。
                                                                                                     * 实时日志流： 审批通过后，自动切换到终端视图，通过 WebSocket 接收 Client 传回的 stdout 流。
                                                                                                     六、 安全与权限控制 (Security)
                                                                                                      * 指令白名单： 即使是 AI 生成的命令，也要经过 Client 端的正则过滤，禁止执行 rm -rf / 等毁灭性指令。
                                                                                                       * 双重鉴权：
                                                                                                          * APP 与网关：基于 JWT 的用户登录态。
                                                                                                             * Client 与网关：基于设备唯一密钥 (Device Key) 的签名认证。
                                                                                                              * 敏感数据脱敏： Client 在上报执行结果给网关前，应自动检测并模糊化日志中的密码、Token 等信息。
